**Date:** 2026-02-28
**Context:** Financial and business review of the v1.9 SaaS plan, plus new additions (admin panel enhancements, Vercel Analytics, landing page design).

## Financial Review Findings

These are corrections and safeguards to apply to the existing v1.9 implementation plan.

### 1. Currency Immutability

- User selects currency during onboarding
- Currency is **locked after onboarding completes** — no change allowed once `isOnboarded: true`
- Account-level `currency` field syncs to user currency, not independently editable
- If a user truly needs to change currency, admin handles it as a manual support case
- Rationale: changing currency after financial data exists corrupts every report, balance, and historical record

### 2. Recurring Transaction Atomicity

- Each recurring transaction execution must be wrapped in its own Prisma `$transaction`
- The transaction creates the Income/Expense record AND advances `nextRunDate` atomically
- If either operation fails, neither commits — no duplicate transactions, no orphaned state
- This prevents the scenario where cron crashes after creating a record but before updating nextRunDate

### 3. CSV Import Safeguards

- **Add `source` field** to Income and Expense models: `MANUAL | IMPORT | RECURRING` (default `MANUAL`)
- **Add `importBatchId`** (nullable String) to Income and Expense — groups imported records for bulk undo
- **Wrap entire import** (createMany + balance update) in a single `$transaction`
- **Post-import budget alerts:** After import completes, run one batch check for affected budgets and send a single summary notification (not per-row)
- **Undo import:** Admin or user can delete all records matching an `importBatchId` in one action, then recalculate balance

### 4. Goal Baseline Amount

- Add `baselineAmount` (Decimal, default 0) to Goal model
- When a goal is linked to an account, `baselineAmount` captures the account balance at the moment of linking
- Goal progress = current account balance - baselineAmount
- If balance drops below baseline, progress shows negative (honest data)
- Unlinked goals use manual contributions, baselineAmount stays 0

### 5. Feature Gating Strategy

- Launch everything open — no gates in v1.9
- Track feature usage per user in admin analytics (adoption rates)
- Feature flags exist for future gating via admin panel
- When ready to introduce tiers: add premium badges, give existing users 30-day grace period
- Rationale: need usage data before knowing which features are worth charging for

## Admin Panel Additions

### Financial Overview Tab (`/admin` dashboard)

**KPI Cards:**
- Total Money Tracked (sum of all account balances across all users)
- Total Transactions (all-time: income + expenses + transfers + payments)
- Transactions This Month
- Average Accounts Per User

**Charts:**
- Platform Growth: line chart — cumulative users + cumulative transactions over time
- Currency Distribution: donut chart — which currencies users selected
- Feature Adoption: horizontal bar — % of users who created at least 1 budget, 1 goal, 1 recurring, 1 import
- Most Used Categories: top 5 expense/income categories across platform

### User Activity Timeline

- Lives in User Detail drawer on `/admin/users`
- Shows last 20 actions chronologically (most recent first)
- **No dedicated audit log table** — query existing models (Income, Expense, Transfer, Budget, Goal) by userId, ordered by createdAt DESC LIMIT 20
- Action types: transaction CRUD, budget CRUD, account created/archived, goal created, CSV import (with row count), login events (from lastLoginAt)

### System Health Monitor (`/admin/system`)

**New model — CronRunLog:**
```prisma
model CronRunLog {
  id             String   @id @default(cuid())
  key            String
  status         String   // "success" | "failed"
  processedCount Int      @default(0)
  errorMessage   String?
  duration       Int?     // milliseconds
  runAt          DateTime @default(now())

  @@index([key, runAt])
  @@map("cron_run_logs")
}
```

**Cron Job Status Panel:**
- Table: each cron key, last run time, result, items processed
- Visual indicators: green (ran within window), yellow (overdue), red (failed)

**Queue Health Panel:**
- BullMQ `getJobCounts()` for each queue: pending, active, failed
- Failed jobs list with error messages and retry button

**Vercel Analytics:**
- Install `@vercel/analytics` package
- Add `<Analytics />` to `app/layout.tsx`
- Web Vitals, page views, referrers handled by Vercel dashboard — no custom admin panel needed

### Data Export

**Per-User Data Export:**
- Button in User Detail drawer: "Export User Data"
- Generates JSON containing: profile, accounts, all transactions, budgets, goals, recurring, notification prefs
- GDPR compliance tool

**Platform Analytics Export:**
- CSV export buttons on admin tables: user list, feature requests, cron logs
- Reuses existing CSV export pattern from user-side income/expense export
