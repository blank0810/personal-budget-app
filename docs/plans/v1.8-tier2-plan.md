## Task A: Dashboard Layout Reorder (#7)

### Problem
The Funds section (Emergency Fund, Savings Goals) is buried at the bottom of the dashboard, below Recent Transactions. From a financial planning hierarchy, wealth-building goals (strategic) should appear before activity logs (operational).

```

2. Secondary Metrics (Credit Utilization + Debt Paydown)
3. Budget Health Summary
4. Recent Transactions (4-col) + Accounts (3-col)   ‚Üê operational

```

### New Layout
```
1. Financial Health Grid (4 KPI cards)
2. Secondary Metrics (Credit Utilization + Debt Paydown)
3. Budget Health Summary
4. Funds Section                                      ‚Üê moved UP (strategic)
```

Move the Funds section JSX block (lines ~875-892) above the Recent Transactions + Accounts grid (lines ~656-873). This is a pure JSX reorder ‚Äî no logic changes.

**Steps:**

## Task B: Income vs Expense Ratio (#6)

### Problem
The existing "Income & Expenses" tab shows absolute amounts (bar chart, pie chart, waterfall) but doesn't visualize the **ratio** ‚Äî the relationship between income and expenses over time. Users can't easily see if they're gaining or losing ground month over month.

### What Exists Already
- `ReportService.getMonthlyComparison()` returns `{ month, income, expense, savings }[]` ‚Äî already fetched on the reports page (line 73)
- `MonthlyComparisonChart` shows income vs expense as side-by-side bars
- `CashFlowWaterfallChart` shows income ‚Üí expense categories ‚Üí net result

### What's Missing
A visualization that shows:
- **Expense-to-Income ratio** as a percentage per month (e.g., "You spent 85% of your income in January")
- **Trend line** showing if the ratio is improving or worsening
- **Savings gap** ‚Äî the actual amount saved each month
- **Color-coded health**: Green (<70% ratio), Yellow (70-90%), Red (>90%), Critical (>100%)

## Task C: Financial Health Check Report (#14)

### Problem
The dashboard shows individual metrics (Net Worth, Savings Rate, Credit Utilization, etc.) but there's no unified diagnostic that says: "Overall, here's your financial health score and here's what to fix."

### Data Available (No New Queries Needed)
All data comes from `DashboardService.getFinancialHealthMetrics()` which already returns:
- `savingsRate`, `ytdIncome`, `ytdExpense`
- `debtToAssetRatio`, `totalDebt`
- `runwayMonths`, `liquidAssets`, `avgMonthlyExpense`
- `creditUtilization`, `totalCreditUsed`, `totalCreditLimit`, `availableCredit`
- `debtPaydown`, `debtPaydownPercent`, `monthsToPayoff`

Plus `DashboardService.getFundHealthMetrics()` for Emergency Fund status.
Plus `ReportService.getMonthlyComparison()` for income vs expense ratio (Task B).

### Design: Weighted Health Score (0-100)

| Pillar | Weight | Grade Logic |
|--------|--------|-------------|
| **Solvency** | 25% | A: Net Worth > 0 & Debt-to-Asset < 30% ‚Üí 100. B: D/A < 50% ‚Üí 80. C: D/A < 75% ‚Üí 60. D: D/A < 100% ‚Üí 40. F: Negative net worth ‚Üí 20 |
| **Liquidity** | 20% | A: Runway ‚â• 6mo (or EF funded) ‚Üí 100. B: ‚â• 3mo (or EF building) ‚Üí 80. C: ‚â• 1mo (or EF underfunded) ‚Üí 60. D: < 1mo ‚Üí 40. F: 0 ‚Üí 20 |
| **Savings** | 20% | A: Rate ‚â• 20% & budget adherence ‚Üí 100. B: ‚â• 10% ‚Üí 80. C: ‚â• 5% ‚Üí 60. D: > 0% ‚Üí 40. F: Negative ‚Üí 20 |
| **Debt Mgmt** | 20% | A: No debt OR utilization < 10% ‚Üí 100. B: < 30% ‚Üí 80. C: < 50% ‚Üí 60. D: < 70% ‚Üí 40. F: ‚â• 70% ‚Üí 20 |
| **Cash Flow** | 15% | A: Expense/Income ratio < 70% ‚Üí 100. B: < 80% ‚Üí 80. C: < 90% ‚Üí 60. D: < 100% ‚Üí 40. F: ‚â• 100% ‚Üí 20 |

**Overall Score** = Weighted average ‚Üí Label:
- 90-100: Excellent
- 75-89: Good
- 60-74: Fair
- 40-59: Needs Attention
- 0-39: Critical

### Implementation

#### Step 1: Create Health Score Service Method
**File:** `server/modules/dashboard/dashboard.service.ts` (MODIFY)

Add method `getFinancialHealthScore(userId: string)`:
- Calls existing `getFinancialHealthMetrics()`, `getFundHealthMetrics()`
- Needs current month income/expense ratio ‚Äî call `getCashFlow()` for current month
- Computes each pillar score using the grade logic table above
- Returns:
  ```typescript
  {
    overallScore: number;      // 0-100
    overallLabel: string;      // "Good", "Fair", etc.
    pillars: {
      name: string;
      score: number;           // 0-100
      grade: string;           // A-F
      weight: number;          // 0.15-0.25
      details: string;         // "Debt-to-Asset ratio is 35%"
      recommendation: string;  // "Reduce debt to below 30% of assets"
    }[];
  }
  ```
