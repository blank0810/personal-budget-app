# Budget Selector Redesign: Drill-Down Navigation

## Problem Statement

The current budget dropdown in the ExpenseForm shows all budgets (current month + 3 months of previous) in a single flat list using `SelectGroup` sections. With 8 budgets/month, this can mean 32+ items visible at once - overwhelming for users. The "Previous Months" section is always visible even though it's only needed for the accrual use case (expense budgeted in January, payment clears in February).

## Design: Drill-Down Popover

Replace the `Select` component with a **Popover-based drill-down selector** that uses page navigation instead of expanding accordion within a flat list.

### User Flow

```
┌─ PAGE 1 (Default) ─────────────────┐
│  ✕  No budget                       │
│  ─── February 2026 ───              │
│  Groceries          (₱5,000 left)   │
│  Transport           (₱2,000 left)  │
│  Entertainment       (₱1,500 left)  │
│                                     │
│  📅 Choose another month  ›         │
└─────────────────────────────────────┘

        User clicks "Choose another month"
        Content slides/replaces →

┌─ PAGE 2 (Month Selection) ──────────┐
│  ← Back                             │
│                                     │
│  ▸ January 2026         5 budgets   │
│  ▸ December 2025        4 budgets   │
│  ▸ November 2025        3 budgets   │
└─────────────────────────────────────┘

        User clicks "January 2026"
        Month expands (accordion) →

┌─ PAGE 2 (Month Expanded) ───────────┐
│  ← Back                             │
│                                     │
│  ▾ January 2026         5 budgets   │
│    Groceries       (₱3,000 left)    │
│    Transport        (₱1,000 left)   │
│    Rent             (₱0 left)       │
│    Insurance        (₱500 left)     │
│    Entertainment    (₱800 left)     │
│  ▸ December 2025        4 budgets   │
│  ▸ November 2025        3 budgets   │
└─────────────────────────────────────┘

        User selects a budget → popover closes
        OR clicks "← Back" → returns to Page 1
```

### UX Enhancements (Lead UI/UX Recommendations)

1. **Selected budget indicator on trigger**: When a previous-month budget is selected, the trigger shows the month in parentheses: `"Groceries (Jan 2026)"` so the user always knows which month the budget belongs to. Current-month budgets show just the name.

2. **Auto-expand on revisit**: If user already selected a January budget, then reopens the dropdown and navigates to Page 2, January should auto-expand so they can see their current selection.

3. **Empty state messaging**: If no previous months have budgets, hide "Choose another month" entirely - don't show it only to reveal an empty page.

4. **Budget count on month rows**: Show `"5 budgets"` next to each month so the user knows what to expect before expanding.

5. **Consistent color-coded remaining**: Same green/amber/red remaining-amount badges from the current implementation, preserved on both pages.

## Component: `BudgetSelector`

New file: `components/modules/expense/BudgetSelector.tsx`

### Why a Separate Component

- The ExpenseForm is already 676 lines - extracting the budget selector reduces complexity
- Follows the same pattern as `CategoryCombobox` (Popover-based, self-contained)
- Encapsulates drill-down state (current page, expanded month) without polluting ExpenseForm

### Props Interface

```typescript
interface BudgetSelectorProps {
  budgets: BudgetWithCategory[];
  value?: string;                    // selected budgetId
  onChange: (budgetId: string | undefined) => void;
  selectedDate?: Date;               // expense date for month grouping
}
```

### Internal State

```typescript
const [open, setOpen] = useState(false);          // popover open
const [view, setView] = useState<'current' | 'months'>('current');  // page
const [expandedMonth, setExpandedMonth] = useState<string | null>(null);  // accordion
```

### Key Behaviors

- **Page navigation**: `view` toggles between `'current'` (Page 1) and `'months'` (Page 2)
- **Reset on close**: When popover closes, reset `view` to `'current'` so next open starts fresh
- **Accordion toggle**: On Page 2, clicking a month toggles `expandedMonth`
- **Auto-expand**: If a previous-month budget is currently selected, auto-set `expandedMonth` to that month when navigating to Page 2
- **Select and close**: Clicking a budget item calls `onChange(budgetId)` and closes the popover
- **"No budget" clears**: Clicking "No budget" calls `onChange(undefined)` and closes
