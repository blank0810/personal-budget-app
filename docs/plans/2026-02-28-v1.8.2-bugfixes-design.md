# v1.8.2 Bug Fixes Design

## Fix 1: Enable liability-to-asset transfers

**Problem:** The transfer form only shows ASSET accounts in the dropdowns. Users cannot transfer between liability and asset accounts (e.g., credit account to cash on hand).

**Root cause:** `TransferForm.tsx` filters the account list to only include accounts with `accountType === 'ASSET'`. The backend service already handles LIABILITY account balance logic correctly.

**Fix:** Remove the ASSET-only filter so all non-archived accounts appear in both "From" and "To" dropdowns.

**File:** `components/modules/transfer/TransferForm.tsx`

## Fix 2: Add comma separators to email currency formatting

**Problem:** Email notifications display currency without comma separators (e.g., `$1234.56` instead of `$1,234.56`), inconsistent with the frontend which uses `Intl.NumberFormat`.

**Root cause:** `notification.service.ts` uses `.toFixed(2)` for currency formatting instead of the shared `formatCurrency` utility.

**Fix:** Import `formatCurrency` from `lib/formatters.ts` and replace all `$${value.toFixed(2)}` occurrences in email templates.

**File:** `server/modules/notification/notification.service.ts`

**Affected locations:**
- Budget alert message text and table (limit, spent, remaining)
- Income notification subject, body text, and table (amount, new balance)
