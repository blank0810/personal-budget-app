# v1.8 Budget Replication Feature - Implementation Plan

## Overview

Allow users to replicate budgets from any past month to a target month, with inline recommendations (increase/decrease suggestions based on history) and selective removal before confirmation.

## User Story

> As a user managing my monthly budgets, I want to quickly replicate my budgets from a previous month to the current/future month, so I don't have to manually recreate 10+ budget entries every month.

## Design Decisions (from brainstorming)

| Decision | Choice |
|----------|--------|
| Source month selection | User picks any past month (flexible) |
| Selective replication | All selected by default, allow removing items before confirm |
| Amount adjustment | Show recommendations inline (increase/decrease based on history) |
| UI placement | Both: button in header + empty state action |

### Phase 1: Backend (Service & Controller)

#### Task 1.1: Add Zod schemas for replication
**File:** `server/modules/budget/budget.types.ts`

Add new types:
```typescript
// Schema for replication input
export const replicateBudgetsSchema = z.object({
  sourceMonth: z.date(),
  targetMonth: z.date(),
  budgetItems: z.array(z.object({
    sourceBudgetId: z.string(),
    name: z.string(),
    amount: z.number().positive(),
    categoryId: z.string(),
  })),
});

export type ReplicateBudgetsInput = z.infer<typeof replicateBudgetsSchema>;

// Response type for getBudgetsForReplication
export interface BudgetReplicationItem {
  id: string;
  name: string;
  amount: number;
  categoryId: string;
  categoryName: string;
  // Recommendation data
  recommendation: 'increase' | 'decrease' | 'stable';
  suggestedAmount: number | null;
  trend: string; // e.g., "Over 3/6 months"
}
```

#### Task 1.2: Add service method to get budgets with recommendations
**File:** `server/modules/budget/budget.service.ts`

Add method `getBudgetsForReplication(userId, sourceMonth)`:
- Fetch budgets for the source month
- For each budget, calculate recommendation using existing `getBudgetRecommendations` logic
- Return `BudgetReplicationItem[]` with recommendation data embedded

```typescript
async getBudgetsForReplication(
  userId: string,
  sourceMonth: Date
): Promise<BudgetReplicationItem[]>
```

**Logic:**
1. Get budgets for source month with category info
2. Get 6-month recommendation data per category
3. Merge recommendation into each budget item
4. Return flat array ready for UI consumption

---

#### Task 1.3: Add service method to replicate budgets
**File:** `server/modules/budget/budget.service.ts`

Add method `replicateBudgets(userId, input: ReplicateBudgetsInput)`:
- Validate target month doesn't already have budgets with same names
- Create new budget records for target month
- Return created budgets or error details for conflicts

```typescript
async replicateBudgets(
  userId: string,
  input: ReplicateBudgetsInput
): Promise<{ success: boolean; created: number; skipped: string[] }>
```

**Logic:**
1. Normalize targetMonth to UTC midnight on 1st
2. Check for existing budgets in target month with matching names
3. Filter out duplicates, track skipped names
4. Batch create remaining budgets using `prisma.budget.createMany`
5. Return summary: created count + skipped budget names

### Phase 2: Frontend (Dialog Component)

#### Task 2.1: Create ReplicateBudgetDialog component
**File:** `components/modules/budget/ReplicateBudgetDialog.tsx`

A dialog component with:
1. **Source month picker** - dropdown of past months that have budgets
2. **Target month picker** - defaults to current month (or next month if current has budgets)
3. **Budget preview table** showing:
   - Checkbox (all checked by default)
   - Budget name
   - Category
   - Current amount
   - Recommendation badge (increase/decrease/stable)
   - Editable amount field (pre-filled with suggested or current)
4. **Summary footer** - "X budgets selected, total: $Y"
5. **Action buttons** - Cancel / Replicate

**Props:**
```typescript
interface ReplicateBudgetDialogProps {
  trigger: React.ReactNode; // Button that opens dialog
  availableMonths: Date[]; // Months with existing budgets
  onSuccess?: () => void;
}
```

**State:**
- `sourceMonth: Date | null`
- `targetMonth: Date`
- `budgetItems: BudgetReplicationItem[]` (fetched when source changes)
- `selectedIds: Set<string>` (checkboxes)
- `amounts: Record<string, number>` (editable amounts)
- `isPending: boolean`
